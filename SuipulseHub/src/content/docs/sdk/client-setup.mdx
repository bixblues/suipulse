# Client Setup

This guide covers how to set up and configure the SuiPulse SDK client in your application.

## Installation

Install the SuiPulse SDK using npm:

```bash
npm install @suipulse/sdk
```

## Basic Setup

### Initialize Client

```typescript
import { SuiPulseClient } from "@suipulse/sdk";

const client = new SuiPulseClient({
  network: "devnet", // or "mainnet"
  // Optional configuration
  options: {
    maxRetries: 3,
    timeout: 5000,
  },
});
```

### Network Configuration

```typescript
const client = new SuiPulseClient({
  network: "devnet",
  rpcUrl: "https://your-custom-rpc-url",
  wsUrl: "wss://your-custom-ws-url",
});
```

## Authentication

### Wallet Integration

```typescript
import { Wallet } from "@mysten/wallet-standard";

const client = new SuiPulseClient({
  network: "devnet",
  wallet: wallet, // Your wallet instance
});
```

### API Key Authentication

```typescript
const client = new SuiPulseClient({
  network: "devnet",
  apiKey: "your-api-key",
});
```

## Error Handling

### Custom Error Handler

```typescript
const client = new SuiPulseClient({
  network: "devnet",
  onError: (error) => {
    console.error("SuiPulse Error:", error);
    // Custom error handling logic
  },
});
```

### Retry Configuration

```typescript
const client = new SuiPulseClient({
  network: "devnet",
  retryConfig: {
    maxRetries: 3,
    retryDelay: 1000,
    retryOn: [408, 500, 502, 503, 504],
  },
});
```

## Best Practices

1. **Environment Configuration**

   - Use environment variables for sensitive data
   - Configure different settings for development and production
   - Keep API keys secure

2. **Error Handling**

   - Implement proper error boundaries
   - Log errors appropriately
   - Provide user-friendly error messages

3. **Performance**
   - Configure appropriate timeouts
   - Use connection pooling
   - Implement caching where appropriate
